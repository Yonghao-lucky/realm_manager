# RealmRouter 配置助手使用指南

**RealmRouter Config Manager** 是一个专为 OpenClaw 用户设计的自动化脚本工具。它可以帮助您一键接入 RealmRouter 的 AI 模型服务，无需手动编辑复杂的 JSON 配置文件。

## ✨ 主要功能

*   **⚡️ 一键接入**：自动注入 RealmRouter 配置，保留原有的 Token 和 Gateway 设置。
*   **🤖 模型切换**：内置 Anthropic, DeepSeek, Google, OpenAI 等主流模型列表，支持按厂商分组切换。
*   **🛡️ 安全备份**：每次修改前自动备份原配置，支持一键还原。
*   **🔑 Key 管理**：支持在线验证 API Key 有效性并快速更换。

---

## 🚀 快速开始

### 方式一：一键安装 (推荐)

在您的终端 (Terminal) 中执行以下命令即可启动工具：

```bash
# 请将 URL 替换为您 GitHub 仓库的实际地址
curl -sSL https://raw.githubusercontent.com/您的用户名/仓库名/main/realm_manager.sh | bash
```

### 方式二：手动运行

如果您已下载了脚本文件：

```bash
# 1. 赋予执行权限
chmod +x realm_manager.sh

# 2. 运行脚本
./realm_manager.sh
```

## 📖 菜单功能说明

脚本启动后会显示如下交互界面，您只需**输入对应的数字编号**并按回车即可操作。

```text
========================================
    RealmRouter 配置管理工具 v2.1
========================================
 [1] 安装/重置 (注入 RealmRouter 配置)
 [2] 更换 Key  (更新 API Key)
 [3] 切换模型  (修改默认 AI 模型)
 [4] 还原备份  (从历史备份恢复)
 [q] 退出

请输入选项编号并回车: 
```

### 1. [安装/重置]
**首次使用请务必执行此步骤。**
*   **操作**：在主菜单输入 `1` 并回车。
*   **流程**：
    1. 提示输入 API Key -> 粘贴您的 Key -> 回车。
    2. 脚本自动验证 Key 有效性。
    3. 验证通过后，自动完成配置注入。

### 2. [更换 Key]
当您的 API Key 过期或需要更换时使用。
*   **操作**：在主菜单输入 `2` 并回车。
*   **流程**：输入新 Key -> 验证 -> 更新。

### 3. [切换模型]
修改 OpenClaw 默认使用的 AI 模型。
*   **操作**：在主菜单输入 `3` 并回车。
*   **流程**：
    1. 进入厂商选择菜单 (如 `[2] DeepSeek`) -> 输入 `2` -> 回车。
    2. 进入模型列表 (如 `[1] DeepSeek-R1`) -> 输入 `1` -> 回车。
    3. 提示切换成功，按回车返回主菜单。

### 4. [还原备份]
如果您发现配置出现问题，可以使用此功能。
*   **操作**：在主菜单输入 `4` 并回车。
*   **流程**：选择列表中显示的备份文件编号 -> 回车确认恢复。

### 5. [更新脚本]
当您在 GitHub 发布了新版本（包含新模型或功能修复）后，用户可以使用此功能自更新。
*   **操作**：在主菜单输入 `5` 并回车。
*   **流程**：脚本自动下载最新版本 -> 覆盖自身 -> 提示重启。

## ⚠️ 重要说明：发布前配置

在您将脚本发布到 GitHub 之前，请务必修改 `realm_manager.sh` 开头的配置项，将 `UPDATE_URL` 设置为您真实的仓库文件地址：

```bash
# 修改 realm_manager.sh 第 9 行
UPDATE_URL="https://raw.githubusercontent.com/您的用户名/仓库名/main/realm_manager.sh"
```

## ❓ 常见问题 (FAQ)

**Q: 脚本支持哪些操作系统？**
A: 支持 macOS 和 Linux 系统。

**Q: 脚本会覆盖我原有的 Token 吗？**
A: 不会。脚本使用增量更新模式，只会添加 RealmRouter 相关的配置，您原有的 Gateway Token、GitHub 插件配置等都会被完美保留。

**Q: 执行时提示 "Python 3 could not be found"？**
A: 脚本依赖 Python 3 来处理 JSON 数据。请确保您的系统已安装 Python 3（通常 macOS 和 Linux 均已预装）。

**Q: API Key 验证失败怎么办？**
A: 请检查您的网络连接，或确认 Key 是否已欠费。脚本允许您选择“强制继续”，但建议您先确保 Key 可用。
